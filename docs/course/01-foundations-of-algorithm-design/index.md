# Module 01 — Foundations of Algorithm Design

## Source scope

This module develops the first course-content block from the syllabus:

> Imperative programming. Algorithm preconditions and postconditions. Forms of algorithm representation. Properties of algorithms. Pseudocode. Basic techniques for structuring algorithms.

**Source allocation for full-time studies:** 1 lecture hour and 2 practical-class hours.

The allocation recorded in the syllabus describes scheduled teaching time. The chapter is intentionally more detailed so that it can also support independent study, revision, and preparation by an instructor.

## Purpose of the module

Before studying searching, sorting, recursion, linked structures, trees, graphs, and complexity, a student needs a precise language for describing a computation.

This module establishes that language. It explains:

- what a computational problem is;
- what distinguishes an algorithm from a program and from one execution;
- how imperative computation changes program state;
- how initial and final assertions specify the required behaviour;
- how one algorithm may be represented in natural language, pseudocode, a diagram, or source code;
- which properties a valid algorithm should have;
- how sequence, selection, iteration, and decomposition structure a solution.

## Learning outcomes

After completing this module, the student should be able to:

1. distinguish a problem, a problem instance, an algorithm, a program, and an execution;
2. identify the input, output, state, operations, and control flow of a simple computation;
3. describe state changes caused by assignments and other statements;
4. formulate a precondition and a postcondition for a simple problem;
5. explain the difference between a specification and an implementation;
6. represent the same algorithm using structured prose, pseudocode, and executable code;
7. identify the essential properties of an algorithm;
8. structure an algorithm using sequence, selection, iteration, and decomposition;
9. trace a simple algorithm manually;
10. identify basic boundary cases, invalid inputs, and side effects;
11. compare equivalent Python and C++ implementations without confusing language syntax with the underlying algorithm.

## Prerequisites

The module assumes only basic familiarity with:

- arithmetic expressions;
- variables;
- simple boolean conditions;
- elementary Python or C++ syntax.

No knowledge of advanced data structures is required.

## How to study this module

For each topic:

1. learn the definitions;
2. inspect the state or execution trace;
3. rewrite the central idea in your own words;
4. compare the pseudocode with Python and C++;
5. answer the self-check questions without looking back;
6. solve the problems;
7. verify that you can explain why the algorithm works, not only reproduce the code.
